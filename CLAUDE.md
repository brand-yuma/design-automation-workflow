# プロジェクト概要
マーケティングクリエイティブ（LP、バナー等）のUIを生成し、Figmaにエクスポートするプロジェクト。

## 技術スタックとコマンド（Web UI生成時）
- **ビルドツール**: Viteを使用すること。
- **テスト**: Vitestを使用すること。
- **よく使うコマンド**:
  - プレビュー起動: `npm run dev`
  - テスト実行: `npm test`

## デザインの基本原則
- **Auto layout必須**: すべての要素はAuto layoutで構成し、レスポンシブに対応させること。絶対配置は禁止。
- **意味のある命名**: レイヤー名は `Frame 1` 等を避け、 `HeroSection` などセマンティックな英語名にすること。
- **アクセシビリティ**: 文字と背景のコントラスト比はWCAG基準（4.5:1以上）を満たすこと。

## ワークフロー
- クライアント別のブランドガイドラインは `clients/【クライアント名】/.claude/rules/brand-guideline.md` を必ず参照すること。
- コードを生成した後、可能であれば `npm test` でエラーがないか確認すること。
- UIの生成が完了したら、必ずFigma MCPを利用してFigmaのキャンバスに編集可能なデザインとして出力すること。
